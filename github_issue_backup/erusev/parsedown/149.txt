  "body": "As this is a \"feature\", I opened this as an issue and not as a PR according to your guidelines.\r\n\r\n----\r\n\r\nSometimes, you want to have inline Markdown in your template.  \r\nA good example is a heading you have already defined in your markup, which is supposed to contain some dynamic texts including links and emphasis.\r\n\r\nYour code would look like this:\r\n\r\n```phtml\r\n<h2 class=\"something\"><?php echo Parsedown::instance()->parse('This is a **cool** heading!'); ?></h2>\r\n```\r\n\r\nUnfortunately, the resulting HTML would be wrapped in a `<p>` tag, not what we want here:\r\n\r\n```html\r\n<h2 class=\"something\"><p>This is a <strong>cool</strong> heading!</p></h2>\r\n```\r\n\r\nI found out that the `parseLine()` method does exactly what we want, so it needs to be public or have an alias like `parseInline($text)`, which can then be used like this:\r\n\r\n```phtml\r\n<h2 class=\"something\"><?php echo Parsedown::instance()->parseInline('This is a **cool** heading!'); ?></h2>\r\n```\r\n\r\nResult:\r\n\r\n```html\r\n<h2 class=\"something\">This is a <strong>cool</strong> heading!</h2>\r\n```",
