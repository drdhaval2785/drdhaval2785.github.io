As a result, Parsedown double encodes HTML entities (see #50).\r\n\r\nInstead, Parsedown should offer a \"safe\" option.\r\n\r\n```php\r\nfunction text($text, $safe_mode = false) {\r\n    $this->safe_mode = $safe_mode;\r\n    if ($safe_mode) {\r\n        $text = htmlentities($text, ENT_QUOTES, 'UTF-8');\r\n    }\r\n    ...\r\n}\r\n```\r\nUsage:\r\n```php\r\n$parsedown = new ParseDown();\r\n$text = \"<script>alert('unsafe text');</script>\";\r\necho $parsedown->text($text, true);\r\n```\r\n\r\nOutput:\r\n```\r\n&lt;script&gt;alert(&quote;unsafe text&quote;);&lt/script&gt;\r\n```\r\n\r\n",
