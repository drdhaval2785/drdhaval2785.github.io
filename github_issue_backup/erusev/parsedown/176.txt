    "body": "@lshaf take a look at the related issue and the discussion there.\r\n\r\nThe markdown spec states\r\n\r\n> If list items are separated by blank lines, Markdown will wrap the items in tags in the HTML output.\r\n\r\nSo\r\n\r\n```markdown\r\n* 1\r\n* 2\r\n```\r\n\r\nshould render to\r\n\r\n```html\r\n<ul>\r\n<li>1</li>\r\n<li>2</li>\r\n</ul>\r\n```\r\n\r\nwhile \r\n\r\n```markdown\r\n* 1\r\n\r\n* 2\r\n```\r\n\r\nshould render to\r\n\r\n```html\r\n<ul>\r\n<li><p>1</p></li>\r\n<li><p>2</p></li>\r\n</ul>\r\n```\r\n\r\nSo far, so clear. Parsedown does not render according to the spec here, because it only adds the paragraph to the top item around the blank line, not both. This is a bug.\r\n\r\nI am not sure about the exact meaning of the spec (seems a bit unclear to me), but I would read it as if\r\n\r\n```markdown\r\n* 1\r\n* 2\r\n\r\n* 3\r\n```\r\n\r\nshould wrap just `2` and `3`."
