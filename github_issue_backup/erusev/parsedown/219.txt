    "body": "@smanzi \r\n\r\n1. MIT license\r\n\r\nThe Expat License, also referred to as the [MIT license is GPL compatible](http://www.gnu.org/licenses/license-list.html#Expat). This means you can distribute MIT licensed code along with GPL licensed code.\r\n\r\nIn your case, if you create a Joomla plugin or extension, your code will need to be licensed GPL, you can however make use of a MIT library like Parsedown and distribute it this along with your plugin or extension. \r\n\r\n2. JED rules\r\n\r\nThe page you are linking to defies the license rules for the JED or Joomla Extension Directory. If you wish to publish and extension on the JED additional rules apply which are more strict. The JED doesn't accept extensions licensed MIT. \r\n\r\nYou can however still publish your extension as GPL which you are required to and distribute Parsedown along side it. This is not a problem at all and is allowed. \r\n\r\nAs Matthew writes [here](http://forum.joomla.org/viewtopic.php?f=262&t=831828&view=previous#p3125047) '...we won't list extensions that are completely MIT licensed...' \r\n\r\nIf you create an extension for Joomla and distribute Parsedown alongside it you are not creating an extension that is completely MIT. Your extension code, even if very simple would be GPL and make use of Parsedown which is MIT. \r\n\r\nThis is perfectly fine for the JED. JED rules only apply if you wish to put your extension on the JED. If you don't then no need to worry about them\r\n\r\nHope that helps. "
