  "body": "I'd like to suggest few cosmetic changes about how Parsedown parses lists:\r\n\r\nIn nested lists, subsequent &lt;ul&gt; tags shouldn't be inline. Right now they appear as &lt;/li&gt;&lt;ul&gt;, that &lt;ul&gt; should be on new line.\r\n\r\nIndentation of &lt;li&gt; with two spaces from its parent &lt;ul&gt;. For nested lists, this should increase, i.e. a second level &lt;ul&gt; should be indented with two spaces from its parent &lt;li&gt; and second level &lt;li&gt; should be indented with two spaces from its parent second level &lt;ul&gt;. Example:\r\n<pre><code>&lt;ul&gt;\r\n--&lt;li&gt;&lt;/li&gt;\r\n--&lt;li&gt;\r\n----&lt;ul&gt;\r\n------&lt;li&gt;&lt;/li&gt;\r\n----&lt;/ul&gt;\r\n--&lt;/li&gt;\r\n--&lt;li&gt;&lt;/li&gt;\r\n&lt;/ul&gt;</code></pre>\r\n\r\nNote: these changes must apply to &lt;ol&gt; as well.\r\n\r\nWhen attempting to add &lt;code&gt; block after &lt;li&gt;, the 4 spaces (or 1 tab) indentation characters are removed. Example:\r\n<pre><code>* this is list\r\n    this is code</code></pre>\r\nwill parse as:\r\n<pre><code>&lt;li&gt;this is list\r\nthis is code&lt;/li&gt;</code></pre>\r\ninstead as:\r\n<pre><code>&lt;li&gt;this is list&lt;/li&gt;\r\n&lt;pre&gt;&lt;code&gt;\r\nthis is code\r\n&lt;/code&gt;&lt;/pre&gt;</code></pre>\r\n- - -\r\nAdditionally, &lt;pre&gt;&lt;code&gt; and their closing tags shouldn't be inline. The code will look nicer and will have better human readability.\r\n",
