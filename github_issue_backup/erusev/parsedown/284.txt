As mentioned on #283, some Markdown editors always insert a title even when it's blank.  In [our project](http://www.cerbweb.com/), we have thousands of knowledgebase articles with these blank titles in the Markdown, which previously worked with PHP Markdown and slightly earlier versions of Parsedown.\r\n\r\nThe issue stems from the regular expression for links that looks for \"one or more characters that aren't a quote\" instead of \"zero or more characters that aren't quotes\".\r\n\r\nSimilarly, it would be perfectly valid in HTML to have ```<a href=\"http://www.example.com\" title=\"\"></a>```, even though it's pointless.\r\n\r\nI added additional tests for blank link titles using single and double quotes, and inline images using blank titles.\r\n",
