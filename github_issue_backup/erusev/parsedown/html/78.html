hkdobrev commented on 2014-02-02 23:21:27

@erusev I've suggested this change in the first place, because I've seen it on other repos. Here are some problems which have happened on other repos:

  • You get PRs for adding a .gitignore file.
    Solution: Close the PR. Link to the GitHub help article.
  • You get PRs with committed system files or vendor/ folder.
    Solution: Ask to remove them from the PR. Link to the GitHub help article.

So because of these scenarios people who use a global .gitignore file (which is still rare), tend to leave a note about it to the eventual contributors.
