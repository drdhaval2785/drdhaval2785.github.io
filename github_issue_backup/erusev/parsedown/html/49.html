erusev commented on 2013-12-18 20:55:19

This is because indented lines that follow list items are considered part of these list items. It is as if they belong to the scope of these list items. This is true for lines of any type. I'm not sure if reference lines should become an exception and get parsed outside of their scope. If they do they won't get parsed in contexts such as the followng:

> [quoted link][1]
>
> [1]: http://example.com
