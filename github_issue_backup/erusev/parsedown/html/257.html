erusev commented on 2015-01-03 17:44:33

Since figure is a block-level element (or isn't a text-level element), it needs to follow the rules for block-level elements in order to get parsed correctly:

The only restrictions are that block-level HTML elements — e.g. <div>, <table>, <pre>, <p>, etc. — must be separated from surrounding content by blank lines, and the start and end tags of the block should not be indented with tabs or spaces. Markdown is smart enough not to add extra (unwanted) <p> tags around HTML block-level tags.

Here's an example that should work:

paragraph

<figure>
    <img attr="value" />
</figure>

paragraph

What is causing the issue in the example quoted in the issue is that the <figure> opening tag starts an HTML block that doesn't get properly closed.
