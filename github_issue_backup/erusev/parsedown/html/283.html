Blank quoted title text in links and inline images doesn't parse properly

jstanden commented on 2015-02-03 21:58:32

Some Markdown editors always insert a title for links and inline images, and just default it to a blank string when the user didn't input anything in the "Title:" prompt. In a perfect world, these editors would just leave off the title argument when content wasn't provided, but this isn't something we can always depend on in our app.

In previous versions of Parsedown, these blank titles parsed properly. In the latest version, the regular expression requires content between the quotes or it doesn't parse the entire link.

The following properly converts to a link:

[link to google](http://www.google.com "This is the link title")

While this doesn't:

[link to google](http://www.google.com "")

This also breaks with inline images that have blank titles:

![blank title](/md.png "")

I've made the minor tweak and added new PHPUnit tests. I'll submit the pull request after submitting this issue.
