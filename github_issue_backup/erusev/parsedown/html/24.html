justdaj commented on 2013-11-04 12:25:46

Hi, sorry I have found a small bug on the latest update if I use multiple divs. Consider the following:

include('inc/Parsedown.php');
$text = 'markdown
<div>
<div><p>some stuff</p></div>
</div>
more markdown';

$result = Parsedown::instance()->parse($text);

echo $result;

This returns

<p>markdown</p>
<div>
<div><p>some stuff</p></div>
<p>&lt;/div>
more markdown</p>

The last DIV is enclosed in a P tag and the < becomes encoded.

I can fix it if I move the final closing DIV up to join the previous line:

 $text = 'markdown
<div>
<div><p>some stuff</p></div></div>
more markdown';

Returns

<p>markdown</p>
<div>
<div><p>some stuff</p></div></div>
<p>more markdown</p>

I can work around it by always ensuring the last DIV is not on its own line, however I thought i should mention it.
