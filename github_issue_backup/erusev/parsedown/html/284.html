jstanden commented on 2015-02-03 22:04:14

This commit fixes an issue where blank titles in links and inline images result in the link not parsing from Markdown into HTML at all. As mentioned on #283, some Markdown editors always insert a title even when it's blank. In our project, we have thousands of knowledgebase articles with these blank titles in the Markdown, which previously worked with PHP Markdown and slightly earlier versions of Parsedown.

The issue stems from the regular expression for links that looks for "one or more characters that aren't a quote" instead of "zero or more characters that aren't quotes".

Similarly, it would be perfectly valid in HTML to have <a href="http://www.example.com" title=""></a>, even though it's pointless.

I added additional tests for blank link titles using single and double quotes, and inline images using blank titles.
