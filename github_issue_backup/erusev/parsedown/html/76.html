Add a CLI mode to allow parsing markdown at the command line

scottchiefbaker commented on 2014-01-30 16:03:07

I was writing a lot of tests, and then copy/pasting the HTML from my browser to the appropriate HTML file, which was very time consuming. Then it occurred to me that we could use Parsedown.php as the CLI tool. With this code you can now do:

php Parsedown.php document.md > document.html

hkdobrev commented on 2014-01-30 16:27:18

BTW why not create a bin/parsedown executable file with a shebang #!/usr/bin/env php. This way there would be no need of CLI checks and any changes to Parsedown.php.

Composer supports binary executables. You could select it via Vendor Binaries feature.

When installing Parsedown with Composer this file would be symlinked in vendor/bin/parsedown.


hkdobrev commented on 2014-01-30 16:28:01

Also there would be no need to use php Parsedown.php. Just ./parsedown.

Adding a bin/parsedown option is easy enough though.
