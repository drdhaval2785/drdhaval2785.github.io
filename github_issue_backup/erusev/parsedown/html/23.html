hkdobrev commented on 2013-11-07 22:49:12

@erusev I think the issue is here: (.*?):

*? look for . any character ) until you find a )

But the ) is not escaped, so some engines might be confused. Perhaps the updated PCRE library would know that a special character as ) cannot exist after *? so it is taking it literally. A more stupider engine would close the group after seeing a ). The group would then end with a *? which is meaningless.


hkdobrev commented on 2013-11-07 22:49:58

Also (.*?) is just before offset 38 in the way I count it.
