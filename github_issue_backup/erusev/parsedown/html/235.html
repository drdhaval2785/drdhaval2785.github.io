hkdobrev commented on 2014-10-09 21:18:27

@markseu I now understand what you were asking in https://github.com/erusev/parsedown/pull/224#issuecomment-58436455. You were actually asking whether this option escapes the & in entities the same way it escapes < in HTML elements.

No, it doesn't. The setMarkupEscaped(true) option is just for markup like inline and block HTML elements and not for HTML entities. Entities would be parsed like in normal Markdown - without modification.

I think you could use \& to show the actual entity code.

I don't think the setMarkupEscaped(true) option should include this.
