Seems related to #173 Only the first list items receives the paragraph, instead of both related items.

Take for example:

* asd

* asd

Will render as

<ul>
<li><p>asd</p></li>
<li>asd</li>
</ul>

Therefore the last list item will never receive a paragraph - which seems like a bug to me.

apfelbox commented on 2014-06-01 15:26:55

@lshaf take a look at the related issue and the discussion there.

The markdown spec states

If list items are separated by blank lines, Markdown will wrap the items in tags in the HTML output.

So

* 1
* 2

should render to

<ul>
<li>1</li>
<li>2</li>
</ul>

while 

* 1

* 2

should render to

<ul>
<li><p>1</p></li>
<li><p>2</p></li>
</ul>

So far, so clear. Parsedown does not render according to the spec here, because it only adds the paragraph to the top item around the blank line, not both. This is a bug.

I am not sure about the exact meaning of the spec (seems a bit unclear to me), but I would read it as if

* 1
* 2

* 3

should wrap just 2 and 3.
