Links and images not parsed if markdown text ends in a list

erangalp commented on 2014-06-21 14:02:30

If the parsed text ends with a list, any links / images in the text are not parsed and instead remain as the markdown representation of the links. Other markdown elements are properly parsed.

Example test case:

##Easy to Customize XCode & Cocos2D Project PLUS High Definition Video Documentation

![img][1]

UPDATE: We added two new game objects: Ladders and Climbing Walls. [See demonstration video][2]

**Features**

 * Complete Source Code for iPhone, iPad and Mac.
 * Two Targets: iOS and Mac and Lite Version Target

  [1]: http://www.commander-cool.com/binpress/commandercool_teaser.png
  [2]: http://www.commander-cool.com/binpress/CommanderCoolLadder.mov

The two links are:

  1. http://www.commander-cool.com/binpress/commandercool_teaser.png
  2. http://www.commander-cool.com/binpress/CommanderCoolLadder.mov

Even though they are in the code block, they disappear in the rendered output on GitHub.
