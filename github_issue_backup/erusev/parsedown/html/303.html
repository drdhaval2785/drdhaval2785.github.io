Make it possible to disable images

ctrlcctrlv commented on 2015-03-29 00:40:08

Hello,

For privacy reasons, I do not want users adding images to markdown. For example, they could add 1px tracking GIF and it would be hard to spot.

So I made a small change to make it possible to be disabled. If you use it make sure to also clean markup because they can still use <img>.


hkdobrev commented on 2015-03-30 21:23:32

@ctrlcctrlv Since this is something which affects only inlineImage and is not something popular it should probably be in an extension. Other options like breaksEnabled and urlsLinked are used to enable/disable certain Markdown flavour-specific features from GFM for instance and markupEscaped is I think a popular request for use in Markdown-enabled user-generated data like comments.

P.S. Actually disabling inline images is suggested by @erusev in the wiki:

You can use the same approach for other elements types. This includes inline element types. For example, to disable Image elements you should override inlineImage.

— https://github.com/erusev/parsedown/wiki/Writing-Extensions#disable-element
