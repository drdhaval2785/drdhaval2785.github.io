wkpark commented on 2014-03-19 10:27:04

The HTML block elements must be preserved. The markdown spec says that all inline tags in the HTML block elements are ignored.

<div>
*inline elements* are ignored.
</div>
paragraph ...

And the Markdown v1.0.2b8(perl)/ Markdown extra(php) add new block HTML elements to include inline elements like as following.

<div markdown="1">
*em **strong*** [link](/url) blah blah
</div>

schucan commented on 2014-03-27 11:47:25

For anyone else. I hacked up a dirty wrapper for this parser. It does two things:

  1. enables the possibility to have <div> elements:
    • [div<any number of attributes>] to start a div container. must be separated on both ends with two newlines.
    • [/div] to end a div container. must be separated on both ends with two newlines.
  2. enables <!-- --> HTML comments

Code:

function getMarkdown ($text) {
  require_once ('/path/to/your/parsedown.php');
  $result = Parsedown::instance()->parse($text);
  if ((mb_strpos($result, '<p>[div') !== false) AND 
      (mb_strpos($result, '<p>[/div]</p>') !== false)) {
    $result = preg_replace('/^\<p\>\[div([^\]]*)\]\<\/p\>/m', '<div$1>', $result);
    $result = str_replace('<p>[/div]</p>', '</div>', $result);
  }
  if ((mb_strpos($result, '&lt;!--') !== false) AND 
      (mb_strpos($result, '-->') !== false)) {
    $result = str_replace('&lt;!--', '<!--', $result);
  }
  return $result;
}
