IEG corrections drawn from faultfinder suggestions

funderburkjim commented on 2015-04-24 20:22:58

This issue is devoted to spelling corrections in the IEG dictionary (Indian Epigraphical Glossary).
There are 427 cases, comprising headwords where:
faultfinder found spelling differences from headwords of MW, and
spelling difference was not of the 'rxx' variety, and
headword appears in sanhw1.txt list only for IEG dictionary,
Since 427 cases is a rather large number to manually check, a further programmatic examination was made.
There are many headwords in IET from languages other than Sanskrit. A random examination of cases
suggested that many of the headwords are likely from the Tamil language. Based on this, a program
separated the 427 cases into two groups:
174 headwords whose entry in IEG contain the word 'tamil'
* 253 other headwords.
The standard two fuzzy alpha files (.txt and .html) were prepared for each of these two subgroups, as
well as for the whole group.






group number txt file html file
non-Tamil 253 txt html
Tamil 174 txt html
All 427 txt html

It is expected that only the first group (non-Tamil) should require examination, and that the Tamil group
should be considered as NO CHANGE without further examination.
It is also certain that some of the non-Tamil group contains words in other languages (English, Greek, Telugu, Persian, maybe others) and will also be marked as NO CHANGE.

The non-Tamil group is ready for someone to adopt - please mention in a comment if you are adopting, so others will
not duplicate your work.

funderburkjim commented on 2015-05-07 19:18:57

This was my first experience using Dhaval's program to separate the modified fuzzyalpha file into 'change' and 'nochange' parts which were then copy pasted
as above.
The php program is quite short, but uses a technique that was new to me.

The file in question (fuzzyalpha.txt) is composed of groups of lines, and the groups are separated by a
line of a certain form (a long bunch of '-' characters). The program reads the entire file into a string blob
(using file_get_contents). Then, it separates this big string into parts based on that special '-------- ... -----'
string. The resulting substrings are thus in fact groups of lines, that we can then do with as we want.
This technique will likely find many other uses. Thanks for posting the program, Dhaval.
