Documenting methods for processing faultfinder errors

drdhaval2785 commented on 2015-04-19 06:04:40

Recently, the method to process the errors (presently faultfinder method, later on extensible to any method) has been changed and more or less fixed now. It uses the suspect word list found by faultfinder method and gives suggestions based on fuzzy logic. It is time to document it for future reference and use.

  1. METHOD TO DEVELOP fuzzyalpha.txt and fuzzyalpha.html @funderburkjim will have to write this.

  2. Download fuzzyalpha.txt and fuzzyalpha.html.
  3. Make a copy of fuzzyalpha.txt and rename it to fuzzybeta.txt (where we will make changes).
  4. Open fuzzybeta.txt and fuzzyalpha.html side by side.
  5. Ctrl+Click on the image links of fuzzyalpha.html to open the images in separate tab.
  6. Verify whether the headword is correct or not.
  7. If the headword is correct, write NO CHANGE in fuzzybeta.txt e.g. 008 aSvatTva -> NO CHANGE
  8. If the headword is incorrect, correct it. The format of correction is wrong word -> correct word : notes e.g. 017 kanzya -> kOzya : The print u is inverted to n.
  9. Once the errors are corrected, save fuzzybeta.txt.
  10. Upload fuzzybeta.txt to github and post its link to github issue where we are tracking errors of particular dictionary.
  11. Make necessary change in the name of dictionary in the code e.g. $dict = "vei"; and run github_issues_post.php.
  12. This separates fuzzybeta.txt in two parts depicting change.txt and nochange.txt.

  13. Copy paste the output of change file into one issue of github titled corrections and no change file to NO CHANGE. e.g. https://github.com/sanskrit-lexicon/CORRECTIONS/issues/98#issuecomment-94237727 and https://github.com/sanskrit-lexicon/CORRECTIONS/issues/98#issuecomment-94237762 respectively.

  14. Integrating the fuzzybeta.txt correction file into database of Cologne. @funderburkjim will have to write this section
