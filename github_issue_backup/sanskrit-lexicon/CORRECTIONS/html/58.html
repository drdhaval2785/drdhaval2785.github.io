re 'Now is not the correct time.'
For these WIL corrections, I partially corrected the entries (by hand) also. The balikarmman headword provides a good example.

Recall that in the digitization, wil.txt for the WIL dictionary, a given headword (such as corrected 'balikarmman') occurs on a particular line of the digitization (118304 in this case):

before current correction of headword:
line 118304 = .{#balikarmBan#}¦ n. ({#-rmBa#}) Offering of oblations to all creatures.
line 118305 = .E. {#bali,#} and {#karmBan#} act.
after current correction of headword:
line 118304 = .{#balikarmman#}¦ n. ({#-rmma#}) Offering of oblations to all creatures.
line 118305 = .E. {#bali,#} and {#karmBan#} act.

In this headword correction process, I always deal only with the line of the digitization that contains the headword, (118304). Normally (i.e., other than for this batch of WIL), I have a small program which does a regex replacement in this headword line of the old headword spelling with the new:

 newline = re.sub('balikarmBan','balikarmman',oldline)

Applying this to line 118304 would give:

line 118304 = .{#balikarmman#}¦ n. ({#-rmBa#}) Offering of oblations to all creatures.

Normally (i.e., except for this batch of Wilson changes) , this would be the final changed form of the digitization at line 118304 (note that 'rmBa' is still wrong).

However, in this batch of Wilson changes, I manually went through the update records and made other changes that I noticed; for this case, I changed 'rmBa' to 'rmma' as shown above.

Note however, that there is still an error in line 118304 that has NOT been corrected.

Doing these extra manual changes does no harm. However, I'll probably not do much more of this now while we are focusing on headwords.
