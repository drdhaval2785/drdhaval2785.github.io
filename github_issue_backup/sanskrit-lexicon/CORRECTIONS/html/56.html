funderburkjim commented on 2014-12-21 00:26:08

Sounds like a good idea.

I think it's best to do this on sandev, since we all have access there.
I copied the SCH/2014 directory there just now.
Start by reading the top-level 'readme.org' file in the SCH directory - just skim read it.

Then look at pywork/update.sh, redo_hw.sh, and redo_xml.sh.
There are a few differences between the operating environment on sandev and that on the Cologne server, but these differences seem minor.
One difference is the 'default' python version. At Cologne, the default version (python --version) is
5.4. However, Cologne also has a 2.6 version, accessible via 'python26'. On sandev, the default version is 2.6, but the 'python26' command is unknown. There are several ways to deal with this on sandev. One way is shown in 'update.sh' , where I put the line

alias python26='python'

at the top.
Another way would have been just to change all 'python26' to 'python' in that file.
sandev is a Linux system. I haven't made this change in other scripts (notably, redo_hw.sh will need this change.)

Instead of Windows batch files (x.bat), you can use shell scripts 'x.sh' and run them via
'sh x.sh'.

I also had to (in the SCH directory)

chmod -R 0755 web

so that the web displays have permission to be displayed.
The 'redo.sh' script in web/sqlite required a minor change, and I had to install a version of 'sqlite3'.
To deal with making 'SAl' the key1 headword instead of 'SAl(ate)', normally (i.e. for most dictionaries) hw1.py would be the program that needs modification. However, SCH is different. And probably a more radical approach will be required, starting with headword.py and hw0.py. It will take some experimenting to get the right approach.
The 'dev' documentation might be good to review (using Cologne site).
I think you will find Python easy to get used to. One important starting detail is that in Python scripts
(like hw1.py), the indentation is significant. Also, there are no ending semicolons, and no curly brackets. I'll explain the other details of Python as questions arise.
