funderburkjim commented on 2014-11-19 02:45:48

In all dictionaries, when there is Devanagari in the print edition, it is coded in a distinguishable way.

In the case of VCP and SKD everything in text is Devanagari.

So, for these cases there are a lot of textual Sanskrit words amenable to spelling checks by faultfinder techniques.

When Sanskrit words are represented in some variant of IAST in the print edition, then, if the dictionary is MW, these Sanskrit words also are identifiable as Sanskrit words, because at some time in the past I marked them with the 

<as0> and <as1> tags.

However, in the case of the other dictionaries, the IAST Sanskrit is coded only as AS (using letter-number combinations for diacritics). There are difficulties in knowing if a given non-Devanagari word in a non-MW dictionary is a Sanskrit word or a word in some other language (English, German). So, this is currently a hard case to handle.
