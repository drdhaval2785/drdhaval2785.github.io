    "body": "@gasyoun \r\n`Why some are in () only?`  All suggestions, whether in parens or no,  are words in some dictionary that are \r\n   within a certain edit distance of the word in question (samudragupwa in case 001 of pgn-fuzzyalpha.txt).\r\n   samudragupta is a word in some dictionary which has a close spelling to samudragupwa,  so \r\n   sumdragupta qualifies as a suggestion.   NOW  imagine we have a list of all suggestions *based on edit\r\n   distance* criterion.   This list is further analyzed into two sublists, based on the preceding and succeeding\r\n   headword  (in case 001 of PGN,  the preceding headword is candragupta, and the following headword is\r\n   also candragupta -  unusual they are the same.)   If a given suggestion is alphabetically between the\r\n   preceeding and following headword, then that suggestion goes in non-parenthesized list;  if the suggestion\r\n   fails to be alphabetically between, then that suggestion goes in the parenthesized list.  \r\n\r\n`Why some () are empty?`    There are no suggestions that are NOT in alphabetical order.  E.g., in case 003\r\n   there is just one suggestion Suklakarmman  and it is alphabetically between ruzwaSarmman and \r\n   suSarmman,  so Suklakarmman   was put outside (), and there was no other suggestion that could go in\r\n   ()\r\n\r\n`Should we have such a big fuzzy distance?`   There is no easy answer to this question.  The edit-distance\r\n  maximum edit distance parameter used in the PGN fuzzyalpha.py program was 6.   This generates some\r\n  false positives (bad suggestions).    I don't know if there is a 'best' edit distance threshold. \r\n"

    "body": "There is two other details of the suggestion generation that is incorporated in fuzzyalpha.py.  \r\n Suppose that the fuzzy distance parameter is set to 6.   And suppose our base word is X (like samudragupwa \r\n is case 001 of PGN).  Our initial pool of candidates for suggestions, Y, is all 300,000+ words in sanhw1.txt.\r\n \r\n* An accepted suggestion Y must have the same first letter as X  (e.g., Y must start with 's')\r\n* Once we have the set S of all words Y which have the same first letter as X and whose edit distance from\r\n   X is <=6,  then this suggestion list is further reduced to include only those Y in S which have the minimum\r\n   edit distance from X.   To elaborate:  Suppose the edit distances from X happen to be 3 for some Y in S,\r\n   4 for some other Y in S, and 6 for other Y in S.   Then the FINAL suggestion list will only include those \r\n   Y with edit distance of 3.\r\n\r\nThe first condition was chosen to provide some computational efficiency.\r\n\r\nThe second condition was chosen as a way to prune the list of suggestions (so that list would not be too long).\r\n\r\nThese choices could be debated,  but, for our purposes in generating correction suggestions they seem to \r\nbe plausible.\r\n"
