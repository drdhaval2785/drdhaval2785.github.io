GitHub Flavored Markdown\r\n\r\nGitHub uses \"GitHub Flavored Markdown,\" or GFM, across the site--in issues, comments, and pull requests. It differs from standard Markdown (SM) in a few significant ways, and adds some additional functionality.\r\n\r\nIf you're not already familiar with Markdown, take a look at Markdown Basics. If you'd like to know more about features that are available in issues, comments, and pull request descriptions, such as task lists, read Writing on GitHub.\r\n\r\n## Differences from traditional Markdown\r\n### Multiple underscores in words\r\n\r\nWhere Markdown transforms underscores (_) into italics, GFM ignores underscores in words, like this:\r\n> wow_great_stuff\r\n> do_this_and_do_that_and_another_thing.\r\n\r\nThis allows code and names with multiple underscores to render properly. To emphasize a portion of a word, use asterisks (*).\r\n\r\n### URL autolinking\r\n\r\nGFM will autolink standard URLs, so if you want to link to a URL (instead of setting link text), you can simply enter the URL and it will be turned into a link to that URL.\r\n\r\nhttp://example.com\r\n\r\nbecomes\r\n\r\nhttp://example.com\r\n\r\n### Strikethrough\r\n\r\nGFM adds syntax to create strikethrough text, which is missing from standard Markdown.\r\n\r\n~~Mistaken text.~~\r\n\r\nbecomes\r\n\r\nMistaken text.\r\n\r\n### Fenced code blocks\r\n\r\nStandard Markdown converts text with four spaces at the beginning of each line into a code block; GFM also supports fenced blocks. Just wrap your code in triple backtics (as shown below) and you won't need to indent it by four spaces. Note that although fenced code blocks don't have to be preceded by a blank line—unlike indented code blocks—we recommend placing a blank line before them to make the raw Markdown easier to read.\r\n\r\nHere's an example:\r\n\r\n\r\nfunction test() {\r\n console.log(\"notice the blank line before this function?\");\r\n}\r\n\r\n\r\nKeep in mind that, within lists, you must indent non-fenced code blocks eight spaces to render them properly.\r\n\r\n### Syntax highlighting\r\n\r\nCode blocks can be taken a step further by adding syntax highlighting. In your fenced block, add an optional language identifier and we'll run it through syntax highlighting.
