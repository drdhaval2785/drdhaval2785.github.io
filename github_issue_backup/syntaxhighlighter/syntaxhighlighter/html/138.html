When I use the build script, the @ABOUT@ variable was not being replaced in my final output file. In digging through the build.js file, I discovered the line:

var files = glob(outputDir, '**.js').concat(glob(outputDir, '**.css'));

When I did a console.log for the value of "files", it returned empty.

I rewrote it as:

var files = glob(outputDir, '**');

The compiled file now does the variable replacement. I realize that it's now processing ALL the files in the output directory, not just the JS and CSS files. I haven't figured out how to do the limiting of files that was intended by the original line yet though.
