elpd commented on 2013-09-05 12:23:26

Check if you have a custom Css with "code" definition in it.

I've Just discovered SyntaxHighlighter and tried to put it to use in my blog on blogger. I've encountered the same problem.

After playing with it I think that my problem arise from a custom Css snippet I am using for previous posts simple code highlights. It's a method I've found on the web describing a way to differentiate your code from the rest of the post specifically for Blogger. A simple background color change and font change.

What I think, from what I gather looking at SyntaxHighlighter script code, is that in the process of the parsing of the post text, the parser captures all the text sections marked for syntaxHighlighter to be processes "pre" and "script" tags ex., and then it divides those section for structural viewing parts and wrap those parts with "code" element tag. The SyntaxHighlighter css defines some rules for those "code" elements and the custom css I've used overrides those. I didn't debug the process but removing the custom Css code relevent to the "code" element worked for me.

Hope this helps.
