s4wny commented on 2013-05-30 20:08:15

Well that is the reason I always remove the !important stuff.

I want to restyle things without using !important to overwrite the SH default styles. !important should rarely be used at all.

For example I often want to do this:

.code-box {
    //Some nice padding
    padding: 10px 10px 0px 10px;
    box-sizing: border-box;

    //Decrease the font-size
    font-size: 0.77em;

    //No scroll bar thx
    word-wrap: break-word;
    overflow: hidden;

    table {
        table-layout: fixed; //Overflow fix
    }
}

(without using !important after every single property)

I think the best solution would be to make SH use specific classes instead of general targeting.

This...

    a,
    div,
    code,
    table,
    table td,
    table tr,
    table tbody,
    table thead,
    table caption,

...should be...

    a.SH-a
    div.SH-div
    code.SH-code
    table.SH-table
    table td.SH-td
    table tr.SH-tr
    table tbody.SH-tbody
    table thead.SH-thead
    table caption.SH-caption

That way peoples existing style sheets won't be a problem and we can drop all !important tags.

Anyway I look forward to the next version of SH.
