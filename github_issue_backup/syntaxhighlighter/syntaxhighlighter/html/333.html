IE8 skips empty lines

davidkokkelink commented on 2014-07-31 14:05:52

In IE8, empty lines are skipped. The total number of lines displayed in the gutter is correct, but the empty line is not being shown (and therefore a new empty line is added as a last line).

I assume this is due to the faulty implementation of String.prototype.split in IE8. splitLines function in shCore.js on line 421:

return block.split(/\r?\n/);

Test: splitLines('abc\n\n') returns ["abc"] in IE8 and ["abc","",""] in newer browsers, so IE8 is skipping two empty lines. It would be better to use XRegExps split function like this:

return XRegExp.split(block, /\r?\n/);

fixed the issue for me.
