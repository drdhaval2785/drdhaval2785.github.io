Call toString() on arguments to RegExp.exec() to avoid error in older IE's

rwstauner commented on 2011-09-26 20:26:34

I found IE 7 and 8 to throw errors... finally traced it back to jQuery calling RegExp.exec on a number.

These older IE's threw a "no method" error when trying to call slice() on a "number" object. putting a toString() on there solves the issue, and stringifying arguments to RegExp.exec seems reasonable to me.

I added tests for XRegExp... not many, but enough to verify the issue.

It looks to me like that particular code block is already an edge case, so I expect this minor workaround should be seldom used and not problematic, but I haven't looked at the rest of the code and you certainly know it better than I do.

Alternatively the argument could be checked before calling toString() on everything... not sure if if(typeof str !== "string")str = str.toString() performs better or makes more sense.

What do you think? Let me know if you think there's something else I need to test.
