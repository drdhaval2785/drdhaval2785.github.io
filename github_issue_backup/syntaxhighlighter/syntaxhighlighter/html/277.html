Extra blank lines inserted

DavidRueter commented on 2013-12-06 23:55:36

Extraneous blank lines are added to the start of the source. Longer source means more blank lines: as many as 51 blank lines or more in my testing.

200 characters of source result in about one blank line, whereas 28,000 characters of source result in 51 blank lines (when Bootstrap is not loaded at all), 20 blank lines (when Bootstrap is loaded) and 12 blank lines (when Bootstrap and Flat-UI-Pro is loaded).

Problem exists with 3.0.83 both from http://alexgorbatchev.com/pub/sh/current/ and built from source. Problem exists on on IE11.0.9600.16428 and FF 25.01.

Problem may exist on other browsers and other versions of SyntaxHighlighter: my testing was limited to this one version and these two browsers.

Adding a global vertical-align: top; to the page corrects the problem--both when Bootstrap is loaded and when it is not. Of course this this may break other things. Also, with Boostrap loaded this does cause the first line to be shifted up about 4 pixels into the green bar at the top of the page).

For example, adding this to the does eliminate the extraneous lines:
<style> * { vertical-align: top; }

</style>
