jeffreytang commented on 2013-02-17 04:02:08

Confirmed, the problem can be reproduced is this example http://jsfiddle.net/gazebotang/AJrS3/3/

when the element is t or l, it will not display correctly. The reason is because the "t" or "l" inside "<" is mistakenly parsed as the element keyword.

It's a bug in line 30 of shBrushXml.js

tag[0].indexOf(tag.name)

when tag.name = "t", indexOf return the index of "t" inside "<", instead of the index of the real element "t".
