rwstauner commented on 2012-11-12 02:58:05

The first problem is a javascript syntax error. Since that fails, SyntaxHighlighter can't find the brush. Comment out this line: https://github.com/chunawall/Syntax_Highlighter_JCL_Brush/blob/master/shBrushJCL.js#L71

Then I was able to get it to work with: <pre class="brush: jcl"> For some reason it doesn't seem to work with brush: JCL. I don't know why, perhaps there's an 'all-uppercase' bug in syntax highlighter, i don't know. But I got it to work with the alias.

My last suggestion would be that some things (browsers, editors, etc) don't like script tags that don't have a closing tag. I would suggest that you use <script src='...' type='text/javascript'></script> rather than <script src='...' type='...'/>. In my experience I've had much less trouble using the former.
