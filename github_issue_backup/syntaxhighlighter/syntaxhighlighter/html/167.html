nmakarov commented on 2012-11-16 02:02:37

... since Bootstrap css tweaks selectors .container:before and .container:after.

As a result, the whole code div shifts about 15 pixels down (related to the gutter) and line numbering becomes thrown out of sync with the code. Confirmed in Firefox 10.0.7 under CentOS.

After some consideration I have replaced the class name from ".container" to ".shcontainer" - two modifications in shCore.js and three in shCore.css.

davidcroda commented on 2012-12-28 16:26:24

Unfortunately, this class is also referenced in the CSS theme files. I was going to look to modify the source to allow you to override the container class name, but since it is hard coded into the CSS I don't think this is feasible.

I think it would make the most sense to change the default to something unique, but this would obviously need to be decided by the maintainer


davidcroda commented on 2012-12-28 16:39:17

I was actually able to fix this without modifying SyntaxHighlighter or Bootstrap by adding the following to my stylesheet. This could probably be integraded into the core without causing issues.

div.syntaxhighlighter .container:before,div.syntaxhighlighter .container:after {
    content:none;
}
