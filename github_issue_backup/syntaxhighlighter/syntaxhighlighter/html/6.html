jmm commented on 2011-09-02 18:58:39

I'm not sure exactly what situation you guys are describing, but in Firefox I was getting an unnecessary and unwanted vertical scrollbar, and even where I wasn't, e.g. in Opera, I found the horizontal scrollbar (which was necessary) unpleasantly close to the highlighted code.

I solved both problems with this kind of thing:

.syntaxhighlighter {

  padding: 1em 0;

}

Also, for unrelated reasons I ended up setting font-size: 0.9em on highlighted code and that solved the problem too, even without padding. But without padding, I still find the horizontal scrollbar too close to the code at the smaller font size.


midimid commented on 2013-01-29 14:45:38

I see this problem in the latest versions of Chrome and Safari, but not Firefox, or IE6-9.

Simply removing the overflow: auto in .syntaxhighlighter of shCore.css works, as does overriding with 1px padding or overflow-y: hidden. I've seen this issue posted on a number of other forums.
