Javascript single-quoted strings with keywords inside messes up the syntax highlighting

github-kiran-user commented on 2013-05-21 17:50:50

Noticed that using the Javascript syntax highlighter does not handle single quoted strings very well.

I have noticed syntax highlighting going haywire for multple cases:

  1. Having a single quote immediately following an open parenthesis - ('
  2. Having a keyword within a single-quoted string messes up the syntax highlighting once the keyword is encountered.

At time it is simply erratic and stops recognizing the string after a while.

This causes strings to be highlighted in reverse - i.e. the end of the string gets treated as the start and vice versa.

russorat commented on 2013-12-14 23:24:35

I found that this is due to the single quote inside the comment, not the parentheses. Since multi line single quotes are rare in Javascript (much rarer than single quotes in comments), I updated scripts/shBrushJScript.js to use the single line versions of the RegExp instead of the multi line. Change lines 18 and 19 to use r.doubleQuotedString and r.singleQuotedString.
