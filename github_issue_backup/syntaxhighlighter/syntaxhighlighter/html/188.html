 blocks into  tags

abbeycode commented on 2013-02-22 16:18:47

I have a code block (below) on a Tumblr blog, which is having its URL turned into a link, even though I define auto-links to false in its <pre> tag, and also in a <script> block.

Code block:

<pre class="brush: applescript; wrap-lines: true; auto-links: false">
    set variable to "URL: http://example.com"
</pre>

I end up with the following text in the rendered page:

set variable to "URL: <a href="http://example.com">http://example.com</a>"

After doing some testing, I saw that this appears to be Tumblr processing the link ahead of time, using its own "auto-links", but the Syntax Highlighter code doesn't know it's not part of the original source. There should be a way to convert tags whose href equals the link text back into straight text, turning <a href="http://example.com">http://example.com</a> back into http://example.com.

Right, if you're showing HTML code that includes links, you would clear that flag. Like the other configuration parameters SH uses, you can override it for one block, in its <class> tag. I also didn't think it would be very common for the link text and URL to match identically. If they don't, the link still passes through as is, even with my flag set (displaying <a href="http://example.com">Example</a>). It doesn't strip all links, just those that appear to be auto generated.
