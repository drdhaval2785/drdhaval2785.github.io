williamrose commented on 2011-11-09 05:44:33

I get a similar issue with the SQL brush, and I think it might be due to unbalanced apostrophes in comments.

For instance, in the following snippet, disobeying the comment and deleting the final &#039; means the keywords in the string literals are highlighted incorrectly.

  <pre class="brush: sql">
  -- Don&#039;t be deleting that final &#039;
  SELECT &#039;Hello, &#039; AS line1
       , &#039;I am going to delete&#039; AS line2
       , &#039;your data now.&#039; AS line3
  FROM dual
  </pre>

If I change all the user's to users in your comments above the syntax highlighting works for me, suggesting this may be the same issue in the JScript brush.
