apollolux commented on 2013-06-29 20:00:02

Same issues here. The cause of the Windows jake problem seems to be the mkdir function in build/build.coffee:37, where dir is initialized to "/" and the path.join loop that comes next prepends it resulting in the extra c:\ that throws the ENOENT. Setting dir to an empty string brings Windows building to the missing components error, but the scss files will still compile to css (which is what I needed at the time).


mctrusty commented on 2013-07-21 12:18:53

Error: ENOENT, no such file or directory '/Users/ken/source/SyntaxHighlighter/components/xregexp/xregexp-all.js' at Object.fs.openSync (fs.js:427:18) (See full trace by running task with --trace)

I was able to fix this by 1.) Installing bower with npm 2.) running bower install to install the packages in bower.json 3.) editing the build.coffee script, line 88, to change:

componentsDir = path.join baseDir, '../components' to componentsDir = path.join baseDir, '../bower_components'

I'm assuming that this is related to this pull request in bower changing 'components' to 'bower_components'

After making these changes, jake ran to completion on bodhi 2.3.0.

On Windows, if I make the change apollolux suggested ("/" -> "") in addition to the above steps, I can get it to run all the way to the validation step, where it errors out on vm.runInNewContext fs.readFileSync(coreFile, "utf8"), context, coreFile.
