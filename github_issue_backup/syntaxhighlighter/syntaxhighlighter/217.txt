    "body": "Well that is the reason I always remove the `!important` stuff.\r\n\r\nI want to restyle things without using `!important` to overwrite the SH default styles. `!important` should rarely be used at all. \r\n\r\nFor example I often want to do this:\r\n\r\n```sass\r\n.code-box {\r\n    //Some nice padding\r\n    padding: 10px 10px 0px 10px;\r\n    box-sizing: border-box;\r\n\r\n    \r\n    //Decrease the font-size\r\n    font-size: 0.77em;\r\n    \r\n\r\n    //No scroll bar thx\r\n    word-wrap: break-word;\r\n    overflow: hidden;\r\n\r\n    table {\r\n        table-layout: fixed; //Overflow fix\r\n    }\r\n}\r\n```\r\n(without using !important after every single property)\r\n\r\n\r\n\r\nI think the best solution would be to make SH use specific classes instead of general targeting.\r\n\r\nThis...\r\n```css\r\n\ta,\r\n\tdiv,\r\n\tcode,\r\n\ttable,\r\n\ttable td,\r\n\ttable tr,\r\n\ttable tbody,\r\n\ttable thead,\r\n\ttable caption,\r\n```\r\n\r\n...should be...\r\n\r\n```css\r\n    a.SH-a\r\n    div.SH-div\r\n    code.SH-code\r\n    table.SH-table\r\n    table td.SH-td\r\n    table tr.SH-tr\r\n    table tbody.SH-tbody\r\n    table thead.SH-thead\r\n    table caption.SH-caption\r\n```\r\n\r\n\r\nThat way peoples existing style sheets won't be a problem *and* we can drop all `!important` tags.\r\n\r\nAnyway I look forward to the next version of SH."
